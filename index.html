Customer Churn Prediction

June 2023 Machine Learning, Python, EDA GitHub Repository

Overview

This project aimed to predict customer churn for a telecommunications company using machine learning techniques. The goal was to identify customers at risk of leaving so that the company could implement retention strategies proactively.

Project Details

The dataset contained information about 7,043 customers with 21 features including account information, services subscribed, and customer demographics. The target variable was a binary indicator of whether the customer churned or not.

Key steps in the project:

  • Exploratory Data Analysis to understand patterns and correlations
  • Feature engineering to create meaningful predictors
  • Handling class imbalance using SMOTE technique
  • Training and evaluating multiple machine learning models
  • Creating interpretable explanations for model predictions

Technologies Used

Python Pandas NumPy Scikit-learn Matplotlib Seaborn KNN SMOTE

Challenges & Solutions

Challenge 1: The dataset had significant class imbalance with only 26.5% churn cases.

Solution: Applied SMOTE (Synthetic Minority Over-sampling Technique) to balance the classes before model training.

Challenge 2: Some features had missing values and outliers that could affect model performance.

Solution: Implemented robust imputation strategies and used tree-based models that are less sensitive to outliers.

Results & Impact

The final KNN model achieved:

  • Accuracy: 93%
  • Precision: 87%
  • Recall: 99%
  • F1-score: 93%

The model helped the company identify high-risk customers with 80% accuracy, enabling targeted retention campaigns that reduced churn by 15% in the following quarter.

Financial Sentiment Analysis

March 2023 NLP, Python, Text Mining GitHub Repository

Overview

This project analyzed sentiment in financial news articles and social media posts to predict market movements. The system classified text into positive, negative, or neutral sentiment to help investors make data-driven decisions.

Project Details

The project involved processing over 10,000 financial documents from various sources including:

  • Financial news articles (Bloomberg, Reuters, WSJ)
  • Earnings call transcripts
  • Analyst reports
  • Tweets from financial experts and companies

Key NLP techniques applied:

  • Text preprocessing (tokenization, lemmatization, stopword removal)
  • Feature extraction (TF-IDF, Word2Vec, BERT embeddings)
  • Sentiment classification using SVM, Random Forest, and Neural Networks
  • Domain-specific lexicon creation with FinBERT

Technologies Used

Python 3.9 NLTK spaCy Gensim Scikit-learn TensorFlow 2.0 Transformers (BERT) Flask API Pandas NumPy

Challenges & Solutions

Challenge 1: Financial language contains many domain-specific terms that standard NLP models don't handle well.

Solution: Created a custom financial lexicon and fine-tuned BERT on financial texts using the Financial PhraseBank dataset.

Challenge 2: Sarcasm and nuanced language in financial commentary.

Solution: Implemented ensemble methods combining lexicon-based and machine learning approaches with attention mechanisms.

Results & Impact

The final model achieved:

  • Accuracy: 87.1% on test set (compared to 72% baseline)
  • F1-score: 86.5%
  • Significantly outperformed standard sentiment analysis tools on financial texts

The system was deployed as a REST API web service that processes real-time financial news, helping hedge funds and individual investors gauge market sentiment more accurately.

House Price Prediction

January 2023 Machine Learning, Regression, Feature Engineering GitHub Repository

Overview

A machine learning model to predict house prices based on various features like location, square footage, number of bedrooms, amenities, and other relevant factors. The project aimed to help both buyers and sellers estimate property values more accurately in the US housing market.

Project Details

The dataset contained information about 20,000 properties with 80 features from Kaggle's House Prices competition. Key steps included:

  • Comprehensive exploratory data analysis with visualization
  • Feature engineering (created new features like age of home, price per sqft)
  • Advanced feature selection using permutation importance
  • Handling missing values and outliers with robust techniques
  • Testing various regression models (Linear, Lasso, Random Forest, XGBoost)
  • Hyperparameter tuning using GridSearchCV
  • Creating an interactive prediction tool with Streamlit

Technologies Used

Python 3.8 Pandas NumPy Scikit-learn XGBoost LightGBM Matplotlib Seaborn Plotly Streamlit Jupyter Notebook

Challenges & Solutions

Challenge 1: Many categorical features with high cardinality (like neighborhood).

Solution: Used target encoding for high-cardinality categorical variables combined with regularization.

Challenge 2: Non-linear relationships between features and target.

Solution: Applied tree-based models (XGBoost) and used partial dependence plots to understand relationships.

Challenge 3: Right-skewed price distribution.

Solution: Applied log transformation to the target variable to normalize distribution.

Results & Impact

The final model achieved:

  • RMSE: $28,500 (12% of average home price)
  • R² score: 0.91 on test set
  • Mean Absolute Error: $21,200
  • Top 15% performance on Kaggle competition

The model was deployed as a web application using Streamlit that allows users to:

  • Input property features through an intuitive interface
  • Get an estimated value with confidence interval
  • See key factors affecting the price through SHAP values
  • Compare with similar properties in the area
